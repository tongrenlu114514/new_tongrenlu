## ADDED Requirements

### Requirement: 用户可以通过网易云歌单ID批量导入专辑

系统 SHALL 允许用户通过提供网易云歌单ID来批量导入该歌单中包含的所有专辑。

#### Scenario: 成功导入歌单专辑

- **WHEN** 用户执行批处理任务并传入有效的歌单ID
- **THEN** 系统获取歌单中所有歌曲
- **AND** 系统从歌曲中提取专辑ID并去重
- **AND** 系统批量导入所有专辑到数据库
- **AND** 系统输出导入统计信息

#### Scenario: 歌单包含大量专辑

- **WHEN** 用户导入的歌单包含超过1000个专辑
- **THEN** 系统分页获取歌单数据
- **AND** 系统逐个处理专辑，不发生内存溢出
- **AND** 系统保存处理进度，支持断点续传

### Requirement: 系统自动跳过已存在的专辑

系统 SHALL 在导入过程中检查专辑是否已存在，自动跳过已存在的专辑。

#### Scenario: 专辑已存在于数据库

- **WHEN** 歌单中的某个专辑已存在于数据库中（通过 cloudMusicId 匹配）
- **THEN** 系统跳过该专辑，不重复导入
- **AND** 系统记录日志说明跳过原因
- **AND** 系统继续处理下一个专辑

#### Scenario: 专辑不存在于数据库

- **WHEN** 歌单中的某个专辑不存在于数据库中
- **THEN** 系统调用专辑详情API获取完整信息
- **AND** 系统保存专辑到数据库
- **AND** 系统保存专辑曲目列表

### Requirement: 系统支持断点续传

系统 SHALL 支持在导入过程中断后恢复处理，避免重复导入。

#### Scenario: 首次执行导入任务

- **WHEN** 用户首次执行歌单导入任务
- **THEN** 系统创建进度文件
- **AND** 系统从第一个专辑开始处理
- **AND** 系统每处理完一个专辑更新进度文件

#### Scenario: 从中断处恢复导入

- **WHEN** 用户重新执行歌单导入任务，且进度文件存在
- **THEN** 系统读取进度文件获取上次中断位置
- **AND** 系统从中断位置继续处理
- **AND** 系统跳过已处理的专辑

### Requirement: 系统提供导入统计信息

系统 SHALL 在导入完成后提供详细的统计信息。

#### Scenario: 显示导入统计

- **WHEN** 歌单导入任务完成
- **THEN** 系统输出以下统计信息：
  - 歌单总专辑数
  - 成功导入数
  - 跳过数（已存在）
  - 失败数
- **AND** 如果有失败的专辑，系统列出失败的专辑ID

### Requirement: 系统正确解析歌单API响应

系统 SHALL 正确解析网易云音乐歌单API返回的数据结构。

#### Scenario: 解析歌单曲目响应

- **WHEN** 系统调用 `/playlist/track/all` API
- **THEN** 系统正确解析 `songs` 数组
- **AND** 系统从每首歌曲中提取 `al.id` 作为专辑ID
- **AND** 系统从每首歌曲中提取 `al.name` 作为专辑名称

#### Scenario: 处理分页响应

- **WHEN** 歌单歌曲数量超过单页限制
- **THEN** 系统使用 limit 和 offset 参数分页获取
- **AND** 系统合并所有页的数据
- **AND** 系统对专辑ID进行去重
