## Context

当前网站首页使用 `particles.js` 实现星空粒子背景效果，具有良好的视觉体验。艺术家数据存储在 `m_artist` 表中，已有 `ArtistService` 支持查询。需要创建一个新的公开页面展示艺术家，使用视觉特效增强浏览体验。

## Goals / Non-Goals

**Goals:**
- 创建视觉吸引力的艺术家展示页面
- 使用动画特效展示艺术家名称
- 支持搜索和点击交互
- 响应式布局

**Non-Goals:**
- 不实现艺术家详情页（点击跳转到音乐库筛选）
- 不新增艺术家管理功能
- 不引入新的前端框架

## Decisions

### 1. 视觉特效方案

**选择**: 浮动气泡/标签云 + 渐变背景

**理由**:
- 艺术家名称以不同大小、颜色的气泡形式浮动
- 复用现有 `particles.js` 作为背景粒子
- 名称大小根据专辑数量动态计算，热门艺术家更显眼
- 纯 CSS 动画实现，无额外依赖

**替代方案**:
- 3D 球形标签云 → 实现复杂，移动端体验差
- 静态网格列表 → 缺少视觉吸引力
- Canvas 粒子 → 性能开销大，SEO 不友好

### 2. 数据加载方案

**选择**: 单次加载全量艺术家

**理由**:
- 艺术家数量预计在数百级别，单次加载可行
- 前端搜索过滤响应更快
- 简化实现复杂度

**替代方案**:
- 分页加载 → 特效展示不便分页
- 虚拟滚动 → 艺术家数量不足以支撑复杂度

### 3. 点击交互方案

**选择**: 点击艺术家跳转到音乐库并自动筛选

**理由**:
- 复用现有音乐库页面
- URL 参数传递筛选条件，支持分享
- 无需新建艺术家详情页

## Risks / Trade-offs

| 风险 | 缓解措施 |
|------|----------|
| 动画性能问题 | 使用 CSS transform + will-change 优化 |
| 艺术家数量过多 | 设置最大显示数量，支持搜索过滤 |
| 移动端触摸区域小 | 气泡最小尺寸 44px，确保可点击 |
| SEO 不友好 | 添加结构化数据，艺术家列表在 HTML 中 |
